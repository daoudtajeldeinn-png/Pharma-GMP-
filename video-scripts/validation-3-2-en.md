# Video Script: Validation - Unit 3.2
## OQ (Operational Qualification)
**Duration:** 20 minutes

---

### Introduction (0:00 - 1:00)
**Speaker:** In this lecture, we will discuss OQ or Operational Qualification. OQ is the second stage of validation and is essential to ensure that equipment operates correctly.

### What is OQ? (1:00 - 5:00)
**Speaker:** OQ is the verification of equipment operation.

**Definition:**
- Operational qualification
- Ensure equipment operates according to specifications
- Ensure all functions operate
- Ensure equipment is safe to operate

**Importance of OQ:**
- Ensures correct operation
- Reduces operational errors
- Ensures measurement accuracy
- Essential before PQ

**Detailed Explanation:**
- OQ must be performed after IQ
- OQ must be fully documented
- OQ is essential before PQ

### OQ Contents (5:00 - 12:00)
**Speaker:** OQ includes several elements that must be verified.

**Main Elements:**

**1. Functional Testing:**
- Test all functions
- Test all modes
- Test all speeds
- Test all parameters

**2. Accuracy Testing:**
- Test measurement accuracy
- Test control accuracy
- Test time accuracy
- Test temperature accuracy

**3. Repeatability Testing:**
- Test result repeatability
- Test result consistency
- Test result stability
- Test result reliability

**4. Safety Testing:**
- Test safety systems
- Test emergency stop
- Test overheat protection
- Test overpressure protection

**5. Limit Testing:**
- Test upper limits
- Test lower limits
- Test critical limits
- Test stability at limits

### OQ Procedures (12:00 - 17:00)
**Speaker:** Executing OQ requires following specific procedures.

**Steps:**

**1. Review Protocol:**
- Review OQ protocol
- Review acceptance criteria
- Review test procedures
- Review required resources

**2. Equipment Preparation:**
- Ensure IQ completion
- Ensure equipment readiness
- Ensure material availability
- Ensure employee training

**3. Execute Tests:**
- Execute functional tests
- Execute accuracy tests
- Execute repeatability tests
- Execute safety tests

**4. Data Collection:**
- Record all data
- Record all observations
- Record any deviations
- Record any problems

**5. Result Analysis:**
- Analyze data
- Compare results with criteria
- Identify any deviations
- Identify any problems

**6. Documentation:**
- Document all activities
- Prepare OQ report
- Review report
- Approve report

### OQ Example (17:00 - 20:00)
**Speaker:** Let's look at a practical example of OQ.

**Example: OQ for a New Mixer:**

**Functional Testing:**
- Test different speeds (50, 100, 150 RPM)
- Test different modes (forward, reverse)
- Test time (5, 10, 15 minutes)
- Test automatic stop

**Accuracy Testing:**
- Test speed accuracy
- Test time accuracy
- Test temperature accuracy

**Repeatability Testing:**
- Execute 3 tests for each speed
- Compare results
- Verify consistency

**Safety Testing:**
- Test emergency stop
- Test overheat protection
- Test overspeed protection

**Documentation:**
- Document all tests
- Prepare OQ report
- Approve report
