# Video Script: CAPA - Unit 3.3
## Documentation and CAPA Closure
**Duration:** 18 minutes

---

### Introduction (0:00 - 1:00)
**Speaker:** In this lecture, we will discuss documentation and CAPA closure. Documentation is essential for regulatory compliance and CAPA closure is essential for effective management.

### Importance of Documentation (1:00 - 5:00)
**Speaker:** Documentation is essential for several vital reasons.

**Main Reasons:**
1. **Regulatory Compliance:** Essential for regulatory inspection
2. **Traceability:** Track all activities and decisions
3. **Training:** Source for training and education
4. **Improvement:** Basis for continuous improvement
5. **Review:** Review performance and effectiveness

**Basic Principles:**
- If not documented, it didn't happen
- Documentation must be accurate
- Documentation must be current
- Documentation must be traceable

### CAPA Documentation Elements (5:00 - 10:00)
**Speaker:** CAPA documentation includes several essential elements.

**Main Elements:**

**1. Problem Description:**
- Describe problem clearly
- Describe problem impact
- Describe problem timing
- Describe problem location

**2. Problem Investigation:**
- Describe investigation
- Describe collected data
- Describe data analysis
- Describe root cause

**3. Actions Taken:**
- Describe corrective actions
- Describe preventive actions
- Describe timeline
- Describe responsibilities

**4. Action Execution:**
- Describe execution
- Describe monitoring
- Describe problems
- Describe solutions

**5. Effectiveness Verification:**
- Describe verification
- Describe results
- Describe effectiveness
- Describe decisions

### CAPA Closure Procedures (10:00 - 14:00)
**Speaker:** CAPA closure requires following specific procedures.

**Procedures:**

**1. Review Effectiveness:**
- Review verification results
- Review action effectiveness
- Review objective achievement
- Review compliance

**2. Make Decision:**
- Make closure decision if effective
- Make re-investigation decision if not effective
- Make decision to develop additional actions
- Make decision to update actions

**3. Close Problem:**
- Close problem in system
- Update records
- Notify concerned parties
- Document

**4. Periodic Review:**
- Review problem periodically
- Monitor non-recurrence
- Update actions when necessary
- Document

### CAPA Records Management (14:00 - 18:00)
**Speaker:** CAPA records management is essential for review and compliance.

**Procedures:**

**1. Create Records:**
- Create record for each CAPA
- Include all required information
- Include all documents
- Include all decisions

**2. Retain Records:**
- Retain records for required period
- Retain records centrally
- Easy access to records
- Protect records from damage

**3. Review Records:**
- Review records periodically
- Review records during inspection
- Review records during problems
- Update records when necessary

**4. Analyze Records:**
- Analyze records to identify trends
- Analyze records to identify recurring problems
- Analyze records to identify improvement areas
- Analyze records to identify improvement opportunities

### Summary (18:00)
**Speaker:** In this lecture, we learned:
- Importance of documentation
- CAPA documentation elements
- CAPA closure procedures
- CAPA records management

**Key Points to Remember:**
- Documentation is essential for compliance
- All CAPA elements must be documented
- Closure is essential for effective management
- Records management is essential for review

In the next lecture, we will discuss CAPA tools.
