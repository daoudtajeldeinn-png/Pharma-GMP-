# سيناريو الفيديو: Validation - الوحدة 4.2
## Computer System Validation
**المدة:** 22 دقيقة

---

### المقدمة (0:00 - 1:00)
**المتحدث:** في هذه المحاضرة، سنتحدث عن Computer System Validation. Computer System Validation ضروري لضمان أن الأنظمة الحاسوبية تعمل بشكل صحيح وموثوق.

### ما هو Computer System Validation؟ (1:00 - 5:00)
**المتحدث:** Computer System Validation هو عملية إثبات أن الأنظمة الحاسوبية تعمل بشكل صحيح وتحقق الغرض المطلوب.

**التعريف:**
- إثبات أن النظام يعمل بشكل صحيح
- إثبات أن النظام موثوق
- إثبات أن النظام آمن
- إثبات أن النظام متوافق مع GMP

**أهمية Computer System Validation:**
- يضمن دقة البيانات
- يضمن أمان البيانات
- يضمن اتساق البيانات
- ضروري للامتثال التنظيمي

**الشرح التفصيلي:**
- Computer System Validation مطلوب من GAMP 5
- Computer System Validation يجب أن يتم قبل الاستخدام
- Computer System Validation يجب أن يُوثق

### معايير GAMP 5 (5:00 - 10:00)
**المتحدث:** GAMP 5 يحدد معايير محددة لـ Computer System Validation.

**المعايير الرئيسية:**

**1. تصنيف الأنظمة (System Classification):**
- Category 1: أنظمة مُدارة (Infrastructure Software)
- Category 3: أنظمة غير مُدارة (Non-configured Software)
- Category 4: أنظمة مُدارة (Configured Software)
- Category 5: أنظمة مخصصة (Custom Software)

**2. دورة حياة النظام (System Lifecycle):**
- Conceptualization
- Project Initiation
- Requirements Definition
- Design
- Development
- Testing
- Deployment
- Operation
- Retirement

**3. إدارة المخاطر (Risk Management):**
- تقييم المخاطر
- إدارة المخاطر
- مراقبة المخاطر
- التحكم في المخاطر

**4. إدارة البيانات (Data Management):**
- أمان البيانات
- سلامة البيانات
- توفر البيانات
- قابلية تتبع البيانات
