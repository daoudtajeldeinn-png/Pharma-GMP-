# Video Script: CAPA - Unit 3.2
## Verifying Effectiveness
**Duration:** 18 minutes

---

### Introduction (0:00 - 1:00)
**Speaker:** In this lecture, we will discuss verifying the effectiveness of actions. Verifying effectiveness is essential to ensure that actions have achieved their objectives.

### Importance of Verifying Effectiveness (1:00 - 5:00)
**Speaker:** Verifying effectiveness is essential for several vital reasons.

**Main Reasons:**
1. **Ensure Solution:** Ensure that the problem is solved
2. **Prevent Recurrence:** Ensure that the problem will not recur
3. **Improve Performance:** Continuous performance improvement
4. **Compliance:** Regulatory compliance
5. **Learning:** Learning from experience

**Detailed Explanation:**
- Verifying effectiveness must be done after execution
- Verifying effectiveness must be comprehensive
- Verifying effectiveness must be documented
- Verifying effectiveness must be reviewed

### Methods for Verifying Effectiveness (5:00 - 10:00)
**Speaker:** There are several methods to verify the effectiveness of actions.

**Main Methods:**

**1. Performance Monitoring:**
- Monitor performance after execution
- Compare performance before and after
- Identify improvements
- Identify problems

**2. Data Analysis:**
- Analyze data after execution
- Compare data with standards
- Identify trends
- Identify effectiveness

**3. Testing:**
- Tests to verify effectiveness
- Tests to verify quality
- Tests to verify compliance
- Tests to verify performance

**4. Review:**
- Review actions
- Review results
- Review effectiveness
- Review improvements

### Effectiveness Criteria (10:00 - 14:00)
**Speaker:** There are several criteria to determine the effectiveness of actions.

**Main Criteria:**

**1. Problem Solution:**
- Is the problem solved?
- Has the problem disappeared?
- Has the condition improved?
- Is the objective achieved?

**2. Preventing Recurrence:**
- Did the problem recur?
- Did similar problems occur?
- Did problem frequency decrease?
- Did performance improve?

**3. Performance Improvement:**
- Did performance improve?
- Did quality improve?
- Did efficiency improve?
- Did effectiveness improve?

**4. Compliance:**
- Is compliance achieved?
- Are regulations followed?
- Are actions documented?
- Are results reviewed?

### Procedures for Verifying Effectiveness (14:00 - 18:00)
**Speaker:** Verifying effectiveness requires following specific procedures.

**Procedures:**

**1. Define Effectiveness Criteria:**
- Define acceptance criteria
- Define performance metrics
- Define timeline
- Define verification methods

**2. Collect Data:**
- Collect data after execution
- Collect data from different sources
- Collect data for a specific period
- Analyze data

**3. Analyze Results:**
- Analyze collected data
- Compare results with criteria
- Determine effectiveness
- Identify improvement areas

**4. Documentation:**
- Document all activities
- Document all results
- Document all decisions
- Document all actions

**5. Closure:**
- Close problem if effective
- Re-investigate if not effective
- Update records
- Document
