# Video Script: GMP Basics - Unit 6.2
## Deviations and Investigation
**Duration:** 22 minutes

---

### Introduction (0:00 - 1:00)
**Speaker:** In this lecture, we will discuss deviations and investigation. Deviations occur in all processes, and investigating them is essential to prevent recurrence.

### What is a Deviation? (1:00 - 5:00)
**Speaker:** A deviation is any departure from established procedures or required standards.

**Definition:**
- Any departure from SOPs
- Any departure from specifications
- Any departure from standards
- Any compliance failure

**Types of Deviations:**
1. **Minor Deviations:** Small deviations that do not affect quality
2. **Major Deviations:** Large deviations that may affect quality
3. **Critical Deviations:** Dangerous deviations that affect patient safety

**Detailed Explanation:**
- Deviations must be reported immediately
- Deviations must be documented
- Deviations must be investigated

### Deviation Investigation Principles (5:00 - 9:00)
**Speaker:** Deviation investigation must follow specific principles.

**Basic Principles:**
1. **Immediate Reporting:** Report deviation immediately
2. **Comprehensive Investigation:** Investigate all possible causes
3. **Complete Documentation:** Document all activities
4. **Corrective Actions:** Develop effective corrective actions
5. **Follow-up:** Monitor corrective action effectiveness

**Detailed Explanation:**
- Immediate reporting reduces damage
- Comprehensive investigation identifies root cause
- Complete documentation provides a record for traceability
- Corrective actions prevent recurrence
- Follow-up ensures effectiveness

### Deviation Investigation Procedures (9:00 - 16:00)
**Speaker:** Deviation investigation includes several organized steps.

**Steps:**

**1. Reporting:**
- Report deviation immediately
- Record deviation details
- Notify direct supervisor
- Notify quality department

**2. Containment:**
- Isolate affected product
- Stop process if necessary
- Protect patients
- Prevent deviation spread

**3. Investigation:**
- Collect evidence
- Interview concerned employees
- Review documents
- Identify root cause

**4. Root Cause Analysis:**
- Use tools like 5 Whys
- Use Fishbone Diagram
- Identify true root cause
- Verify root cause

**5. Corrective Actions:**
- Develop corrective actions
- Implement actions
- Evaluate effectiveness
- Document

**6. Follow-up:**
- Monitor action effectiveness
- Re-investigate when necessary
- Update procedures
- Document

**Detailed Explanation:**
- Reporting must be immediate
- Containment is essential to protect patients
- Investigation must be comprehensive
- Root cause analysis is essential to prevent recurrence
- Corrective actions must be effective
- Follow-up is essential to ensure effectiveness

### Root Cause Analysis Tools (16:00 - 22:00)
**Speaker:** There are several tools that can be used for root cause analysis.

**Main Tools:**

**1. 5 Whys:**
- Ask "why" 5 times
- Identify root cause
- Simple and easy to use
- Effective for simple problems

**Example:**
- Why did the deviation occur?
- Why did the cause occur?
- Why did the second cause occur?
- Continue until reaching root cause

**2. Fishbone Diagram (Ishikawa):**
- Visualize all possible causes
- Categorize causes into groups
- Effective for complex problems
- Provides comprehensive view

**Main Categories:**
- Materials
- Machines
- Manpower
- Methods
- Environment
- Measurement

**3. Pareto Analysis:**
- Identify most common causes
- Focus on main causes
- Effective for recurring problems
- Helps determine priorities

**Detailed Explanation:**
- 5 Whys is simple and fast
- Fishbone is comprehensive and organized
- Pareto helps determine priorities
- Multiple tools can be used together
