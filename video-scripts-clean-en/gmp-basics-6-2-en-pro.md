Deviations occur in all processes, and investigating them is essential to prevent recurrence.
What is a Deviation?
(1:00 - 5:00) A deviation is any departure from established procedures or required standards.
Any departure from SOPs Any departure from specifications Any departure from standards Any compliance failure Types of Deviations: Minor Deviations: Small deviations that do not affect quality Major Deviations: Large deviations that may affect quality Critical Deviations: Dangerous deviations that affect patient safety Deviations must be reported immediately Deviations must be documented Deviations must be investigated Deviation Investigation Principles (5:00 - 9:00) Deviation investigation must follow specific principles.
Immediate Reporting: Report deviation immediately Comprehensive Investigation: Investigate all possible causes Complete Documentation: Document all activities Corrective Actions: Develop effective corrective actions Follow-up: Monitor corrective action effectiveness Immediate reporting reduces damage Comprehensive investigation identifies root cause Complete documentation provides a record for traceability Corrective actions prevent recurrence Follow-up ensures effectiveness Deviation Investigation Procedures (9:00 - 16:00) Deviation investigation includes several organized steps.
Steps: Reporting: Report deviation immediately Record deviation details Notify direct supervisor Notify quality department Containment: Isolate affected product Stop process if necessary Protect patients Prevent deviation spread Investigation: Collect evidence Interview concerned employees Review documents Identify root cause Root Cause Analysis: Use tools like 5 Whys Use Fishbone Diagram Identify true root cause Verify root cause Corrective Actions: Develop corrective actions Implement actions Evaluate effectiveness Document Follow-up: Monitor action effectiveness Re-investigate when necessary Update procedures Document Reporting must be immediate Containment is essential to protect patients Investigation must be comprehensive Root cause analysis is essential to prevent recurrence Corrective actions must be effective Follow-up is essential to ensure effectiveness Root Cause Analysis Tools (16:00 - 22:00) There are several tools that can be used for root cause analysis.
Main Tools: 5 Whys: Ask "why" 5 times Identify root cause Simple and easy to use Effective for simple problems Example: Why did the deviation occur?
Why did the cause occur?
Why did the second cause occur?
Continue until reaching root cause Fishbone Diagram (Ishikawa): Visualize all possible causes Categorize causes into groups Effective for complex problems Provides comprehensive view Main Categories: Materials Machines Manpower Methods Environment Measurement Pareto Analysis: Identify most common causes Focus on main causes Effective for recurring problems Helps determine priorities 5 Whys is simple and fast Fishbone is comprehensive and organized Pareto helps determine priorities Multiple tools can be used together (22:00) In this lecture, we learned: What is a deviation Deviation investigation principles Deviation investigation procedures Root cause analysis tools Key Points to Remember: Deviations must be reported immediately Comprehensive investigation is essential Root cause analysis prevents recurrence Corrective actions are essential In the next lecture, we will discuss complaints and recalls.
